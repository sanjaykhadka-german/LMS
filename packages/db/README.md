# @tracey/db

Drizzle ORM schema and client for Tracey. All Tracey-side tables live in the Postgres `app` schema, separate from the Flask LMS's `public` schema.

## Tables

- `app.tenants` — one row per Clerk organisation; the multi-tenant primitive.
- `app.processed_stripe_events` — Stripe webhook idempotency table keyed on `event.id`.

## Scripts

- `pnpm db:generate` — runs `drizzle-kit generate` to diff `src/schema.ts` against the latest snapshot and emit a new SQL migration. Run this whenever you change the schema.
- `pnpm db:migrate` — applies any pending migrations to the database in `DATABASE_URL`. Idempotent. Wired into `lms-web`'s Render build step so production gets migrated on every deploy.
- `pnpm db:studio` — opens Drizzle Studio (local UI for the `app` schema only — does not show Flask's `public` tables).

## Notes

- The migrator creates the `app` schema and the `pgcrypto` extension (needed for `gen_random_uuid()`) before running migrations.
- A meta snapshot is regenerated by `drizzle-kit generate` on first run after install. The committed `0000_initial_app_schema.sql` + `_journal.json` are sufficient for `db:migrate` to bring up a fresh database without running `generate` first.
